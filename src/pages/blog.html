Javascript is Weird

8:57pm 09-11-2016

I am currently studying at Founders and Coders in London. It is a web development bootcamp that focuses on Javascript and Node JS.

During the course of my studies I came across some behaviour in Javascript that I found to be counterintuitive, and which demonstrates why a solid understanding of how your programming language works is important.

Look at the two code snippets below. I assumed that they would behave identically. In reality only one of them works correctly. Can you spot which one will work?

example two



If you said example one, you are correct!

These two examples are different because when determining which variables a function has access to, the position where the function is declared is more important than the position from which they are called.

Because the callback function in example 2 is declared outside of the forEach array method, it is unable to access the index function which it needs to work correctly. In example 1 because we have declared our callback function within the forEach array method, we can use the index variable happily.

I had assumed that being called inside the forEach method, our callback function would be able to use the index variable, but this is not the case. Lexical scoping is hugely important in javascript, and Iâ€™m glad to have learned this lesson now, rather than whilst working on production code!
